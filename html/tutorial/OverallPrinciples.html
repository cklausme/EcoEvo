ECOEVO PACKAGE TUTORIAL

Overall Principles

The purpose of the EcoEvo package is to streamline the analysis of ecological and eco-evolutionary models. Some of the design principles are described below.

General

Once a model is set for analysis with SetModel, equations don't need to be re-entered as arguments of each function.

The same functions work on different model types (ContinuousTime, DiscreteTime, periodic, etc.).

Rule lists are used as both input to and output of various functions (e.g. as initial conditions and as equilibria).

Simulation output is in the form of temporal rule lists, which can be plotted with PlotDynamics, used as input to various functions (in which case they are appropriately averaged), or turned into numerical rule lists using Slice and FinalSlice.

Many functions will work analytically if numerical values aren't given.

Functions that start with Solve use Solve, those that start with NSolve use NSolve, and those that start with Find use FindRoot or possibly FixedPoint.

Options

Most functions take optional arguments at the end of the function call.

Functions that call other functions can pass options using *Opts. E.g. EcoSim calls NDSolve, so to pass options to NDSolve, use NDSolveOptsndsolveoptions in EcoSim. These can be nested.

Most functions have the option Verbose. Setting VerboseTrue prints out extra information about what the function is actually doing. Setting VerboseAllTrue sets Verbose for all functions within.

Functions with multiple methods have the option Method.

Potentially time-consuming numerical functions have the option Monitor to show a progress monitor.

Many functions have the option Fixed, which fixes variables at the given values.

Populations

Populations that are omitted in initial values are assumed to be Fixed at zero.

Guilds

Guilds are open-ended with respect to the number of species.

When setting up model equations, guild is a placeholder for the number of species in guild guild.  (script N) can be entered with the shortcut -scN-.

Species number is denoted using subscripts (ctrl-minus). E.g. {x11,x22}

The number of species in a particular function call is inferred from the traits and/or population values given. E.g. using the traits defined above in a function would temporarily get guild equal to 2.

When setting up traits or populations, guildn makes a "blank" set of n species in guild guild. E.g. trait specification {guild2} would be treated as {x1x1,x2x2} if there was a single trait x and population specification {guild2} would be treated as {n1n1,n2n2}.

Functions that take both traits and population rule lists can take a combined rule list of traits and population values. E.g. {x11,n11} would be split into {x1 1},{n11}.
