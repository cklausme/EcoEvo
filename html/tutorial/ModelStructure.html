Model Structure

There are three types of entities in an EcoEvo model:

Auxiliary variablesAbiotic variables
PopulationsBiotic variables, possibly structured into multiple components
GuildsLike populations, but with an open-ended number of species and possibly with one or more traits

In addition, ModelType defines the type of model — ContinuousTime (default) or DiscreteTime — and Period give the period of external forcing – ∞ for an unforced model (default), a positive real number for a periodically forced model, and 0 for an aperiodically forced one.

The active model to be studied is set with SetModel and can be unset with UnsetModel. Information on the current model can be seen with ModelInfo.

Auxiliary variables

Auxiliary variables are for abiotic variables such as nutrients. Each auxiliary variable requires a Variable and an Equation, and optionally accepts a Range (default = Interval[{0,∞}]).

Populations

Populations are for biological species. They can consist of one or more Components, each of which requires a Variable and an Equation, and optionally accepts a Range (default = Interval[{0,∞}]). If there is only one component, it doesn't need to be specified explicitly. Components have a Type, which can be either "Extensive" (default) for number of individuals or "Intensive" (measured on a per individual basis). Invasion rate can be calculated for populations with Inv and unspecified populations are assumed to be equal to zero in EcoSim, SolveEcoEq, NSolveEcoEq, and FindEcoEq.

Equations can be defined in place or with variables.

Guilds

Guilds are for sets of species subject to evolution or community assembly, that all share the same fitness-generating function (sensu Vincent & Brown 2005) They have unstructured or structured components as populations have, but also can have a number of traits.
