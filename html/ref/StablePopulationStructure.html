StablePopulationStructure

StablePopulationStructure[pops, inv]
calculates the stable population structure of invader inv invading the resident community pops.

StablePopulationStructure[traits, pops, Guildguild]
calculates the stable population structure of an invader in guild guild (default=first), using resident trait values traits.

StablePopulationStructure[traits, pops, traitinv]
uses invader traits traitinv.

DetailsDetails

  • The identity of the invader is specified as a variable (for unstructured pops), as a name (for structured pops), as a Guild (first guild if omitted), or as a list of traits.
  • When a Guild is given, the invader traits are denoted with subscript 0.
  • Based on the model type and inputs, StablePopulationStructure has three modes: "Eigenvalue", "ContinuousTime Floquet", and "DiscreteTime Floquet". Currently only "Eigenvalue" mode is compatible with StablePopulationStructure.
  • "Eigenvalue"(model is unforced [Period = 0] AND pops is not a function of time) OR (model is forced [Period != 0] AND (Method = "Instantaneous" OR Time is given))
    "ContinuousTime Floquet"ModelType is "ContinousTime" AND ((model is forced [Period != 0] AND Method != "Instantaneous" AND Time is not given) OR pops is a function of time)
    "DiscreteTime Floquet"ModelType is "DiscreteTime" AND ((model is forced [Period != 0] AND Method != "Instantaneous" AND Time is not given) OR pops is a function of time)
  • "Eigenvalue" mode first solves for quasi-steady state equilibria of intensive components (if any), then plugs these into equations of extensive components, which are assumed to be zero. In the case of only one extensive component, stable population structure is simply the quasi-steady state intensive components. In the case of more than one extensive components, stable population structure also includes the right eigenvector corresponding to the dominant eigenvalue of the Jacobian matrix of extensive component equations.
  • The following options can be given:
  • Method"Automatic"method to calculate invasion rate
    Timetfix time if nonautonomous model
    SolveOpts{}options to pass to Solve
    NSolveOpts{}options to pass toNSolve
    FindRootOpts{}options to pass to FindRoot
    EigensystemOpts{}options to pass to Eigensystem
    QSSMethod"NSolve"method to solve for intensive components in quasi-stable state
    QSSICs"Automatic"initial guess to solve for intensive components in quasi-stable state
    SimplifyResultFalsehow to simplify results
    Guild"Automatic"guild invader
    VerboseFalseprint out extra info
    VerboseAllFalseapply Verbose recursively
